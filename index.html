Thrill is a test runner for teams needing to test their web application on many browsers.

Browsers connect to a central server. Developers requests tests to be run on the connected browsers remotely. For most projects, the only thing a developer will need to know to run tests is the host of the central server.

Here are some features you might find... thrilling:

  • Automatic adaptation of QUnit, jasmine, mocha, or YUI tests, allowing you to run them without any additional configuration.
  • Targeting of testing browsers based on user-agent (browser type), or capabilities (anything Modernizr can detect).
  • Automatic recovery of unresponsive browsers.
  • Automatic population of the central server using a Selenium Server, or from the cloud via Sauce Labs or Browser Stack.
  • Ability to run tests through the command like, as a Grunt task, or within your own library.
  • A simple API to write your own adapters or reporters.
  • Ability to use mocha's reporters (even when you're running, say, QUnit tests).
